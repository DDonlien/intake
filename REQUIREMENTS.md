# Intake 需求追踪

Intake 是一款 iOS 饮食摄入与实时热量预算 App。产品通过 Apple 健康同步身体数据、静息消耗、活动消耗和运动数据，结合营养数据库、手动饮食记录、语音输入与 AI 拍照识别，帮助用户管理每日热量预算、蛋白质目标和长期饮食趋势。

## 阶段 0：产品基础与范围定义

### 产品定位

- [ ] [FOUND-A-000] 定义 Intake 产品基础 #epic #P0
  - [ ] [FOUND-A-001] 定义产品名称与描述
    - [ ] 产品名称为 `Intake`
    - [ ] 一句话描述为 `AI-powered nutrition budget for everyday meals.`
    - [ ] 对外定位为 iOS 饮食摄入、热量预算与营养目标追踪 App
  - [ ] [FOUND-A-002] 定义产品核心定位
    - [ ] Intake 不是传统饮食记账本
    - [ ] Intake 的核心是实时热量预算
    - [ ] Intake 关注当天还能吃多少、已经吃了多少、当前缺口是多少
    - [ ] Intake 同时关注蛋白质目标，不只关注卡路里
  - [ ] [FOUND-A-003] 定义产品家族关系
    - [ ] Intake 与 Pulse 属于同一产品家族
    - [ ] Intake 继承 Pulse 的 iOS 原生 Liquid Glass 视觉方向
    - [ ] Intake 默认使用浅色模式
    - [ ] Intake 保留深色模式扩展能力
  - [ ] [FOUND-A-004] 定义核心用户问题
    - [ ] 用户可以知道某一天已经吃了多少
    - [ ] 用户可以知道某一天当前热量缺口是多少
    - [ ] 用户可以知道某一天还能吃多少
    - [ ] 用户可以知道运动后当天目标上限是否变化
    - [ ] 用户可以知道蛋白质和宏量营养目标完成情况
    - [ ] 用户可以快速记录一顿饭
    - [ ] 用户可以回看过去几天的饮食记录

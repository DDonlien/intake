1. Bank 页底部有一个可输入的 `TextField` "Search foods, meals, brands"

3. 关闭键盘后，缩放状态偶发不还原
4. 用户希望直接禁用这种缩放

排查 `Intake/Views/BankView.swift` 源后发现：源里的搜索条仍是**纯静态 HStack**（图标 + 占位 Text，无 `TextField`），与截图里的可输入版本完全对不上号——用户跑的 build 和源脱节。`Intake/` 下也没有任何 `TextField` / `searchable` / `@FocusState` 出现。

结合 `REQUIREMENTS.md` 里 v0.1.0 已经写死的「Bank 搜索与筛选 Tab 移到底部操作区，悬浮锁定」要求 (`0.1.0-UI-A-050/051`)，可以判断用户本地 build 已经在向那个方向走，但源没跟上。所以本次任务实际是：

1. 把 Bank 页结构对齐到 v0.1.0 设计（搜索 + 筛选底部悬浮锁定）
2. 把静态占位条升级为真正的 `TextField`
3. 修 iOS 自动放大问题（根因：TextField 字体 < 16pt）
4. 修"退出文字输入缩放不消失"（根因：键盘 inset 残留 + 自动放大未撤销）

## 对话与行动记录

### 1. 现状摸查

**iOS 端相机权限 = 完全没用**：
- Info.plist 没有任何 `NS*UsageDescription` 键
- 整个 swift 代码搜不到 `AVCaptureDevice` / `UIImagePickerController` / `PHPicker`
- RecordView 里所有按钮 `Button(action: {})` 是空闭包，包括相机快门
- "Point camera at your meal" 是占位 emoji 行，没有 AVCaptureSession 也没有任何真预览

**结论**：用户说"相机权限现在没用"完全准确。Info.plist 没 key + 代码没用 camera API = 系统永远不会弹权限框。

### 2. 最小可用接入方案

为符合 iOS 26 + Liquid Glass 风格、又能稳运行，选了：
- **Camera**：封装 `UIImagePickerController(sourceType: .camera)` 为 SwiftUI `UIViewControllerRepresentable`（`CameraCaptureSheet`）——iOS 26 没有原生 SwiftUI `CameraPicker`，这是最稳的桥接
- **Library**：直接用 SwiftUI 原生 `PhotosPicker`（iOS 16+，无权限弹框，读相册不需要 NSPhotoLibraryUsageDescription）
- **权限流**：`AVCaptureDevice.requestAccess(for: .video)` async API，denied 时弹 alert 引导去 Settings
- **预览**：相机区有图就显示图 + retake 按钮（`xmark.circle.fill` 角标），没图就显示原本的 placeholder

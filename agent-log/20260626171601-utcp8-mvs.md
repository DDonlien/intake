## 对话与行动记录

### 1. 现状摸查

**1.1 Server 端校验实际工作**
直接 curl 测生产端三个关键场景：
```
注册已存在邮箱 → 400 "This email is already registered."
登录未注册邮箱 → 401 "Email not found. Register first."
登录错误密码   → 401 "Wrong password."
```
Server 端校验是**正常的**。客户端生产 bundle 也确实有 `Email not found` / `Wrong password` 字面量。

**1.2 "所有人看到同一套数据"的真相**
Playwright 跑通多账号实测：
- 注册 A，登录后写入 `intake.data.3d2e2c11-...-743ed18f0fdf.v1`，内容是 `makeDefaultData()`（Tao/78.4kg/2334 kcal/14 个默认 foods）
- 登出，注册 B，登录后写入 `intake.data.3259aa3d-...-470d3ef299ff.v1`，内容**完全相同**

storage **是**按 userId 隔离的（key 不同），但所有新账户跑同一份 `makeDefaultData()` 硬编码默认值，所以**视觉上**"所有人看到同一份数据"。

**1.3 "账号权限校验缺失"为什么体感是真的**
- 客户端缺少 accountId 内嵌字段，无法独立审计数据归属
- 客户端缺少"当前是哪个邮箱"的醒目展示
- 用户登录后没有任何"我刚登录的是这个账号"的二次确认

### 2. 修复方案

不只补 server 校验（已 OK），**重点改客户端**：

1. **`AppData` 新增 `accountId` 字段**
   - 类型上强制约束
   - 保存时强制 stamp 当前 `account.id`（防止 account 切换时写入到错 key）
   - 加载时校验：如果 `stored.accountId !== currentAccount.id` → 视为外来数据 → 初始化为空
   - 老数据（无 `accountId`）首次加载时自动 stamp 当前账号，**保留原内容**（迁移友好）

2. **新账户从 `makeEmptyData()` 启动**
   - 不再注入 demo 数据
   - 满足 `[0.2.0-DB-A-019]` "新账户首次登录时创建空饮食历史或明确的示例导入选择"

3. **新增 "Load demo data" 入口**
   - MePage Settings 增加按钮，复制 `Reset demo data` 文案
   - 想看示例的用户主动点击

4. **UI 顶部显示当前登录账号**
   - profile-card 用蓝色 pill 展示 "Signed in as {email}"
   - avatar fallback 用 email 首字母
   - profile.name 为空时也用 email 显示

5. **`normalizeData` 区分空 profile vs 有 profile**
   - 空 profile（name=""、weight=0、height=0、age=0）→ goals 全零
   - 否则按 plan preset 算 goals

### 3. 验证

用 Playwright 在 dev mode 跑了 4 个场景：

| 场景 | 期望 | 实际 |
|------|------|------|

| Load demo data | 主动加载 demo 后内容 = Tao/78.4kg/14 foods | ✓ confirm 弹出，数据正确切换 |

## 完成工作

### 1. 数据存储层（DataStore）

- `src/utils/dataStore.ts` — 完整的 CRUD + 持久化
- 默认新用户：空 user、空 dates、空 meals
- 支持 `loadSampleData()` 加载演示数据
- `clearAll()` 清空所有数据并删除 localStorage
- 数据通过 `getUser()` / `getDates()` / `getMeals()` 提供给 UI

### 2. 空状态 UI

| 区域 | 空状态 | 有数据 |
|------|--------|--------|
| ME 头像 | "新用户" | 真实名字 |
| 目标摘要 | 全部 "--" | 真实值 |
| 计划 | 全部 "--" | 真实值 |
| 能量环 | "--" | 真实值 |
| 进度条 | 0% | 真实百分比 |
| 健康同步 | "-- kcal" | 真实值 |
| 日期条 | 只有"今天" | 历史记录 |
| 餐次 | "还没有记录" | 食物列表 |

### 3. "清空个人数据"按钮

- 位置：ME 页面底部，和健康数据卡片同级的列表行
- 样式：红色图标 + 红色标题 + 红色箭头
- 行为：点击弹出 `confirm()` 确认对话框
- 确认后：调用 `DataStore.clearAll()`，触发整页重新渲染
- 清空后：页面恢复空状态，进度页显示 "--"

### 4. "加载演示数据"按钮

- 位置：空状态 ME 页面底部
- 用途：方便测试，一键加载 v6 的完整演示数据
